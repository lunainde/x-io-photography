# X-iO — Photography & Film, Berlin

The X-iO photography/video portfolio site. Built with Next.js and Sanity CMS.

The actual app lives in **[`site/`](site/)** — all commands below are run from
inside that folder.

## Running it locally

```bash
cd site
npm install
npm run dev
```

Then open **http://localhost:3000**.

## Managing images and videos

Content is managed in **Sanity Studio**, embedded right in the app at
**http://localhost:3000/studio** (sign in with the account used to create the
Sanity project). No separate tool or login needed.

### Add, edit, or delete a single photo/video

1. Open `/studio`.
2. Click **Media item** in the left sidebar to see everything currently live.
3. To add: click **+** (top right), fill in:
   - **Title** — internal label only, not shown on the site
   - **Categories** — pick one or more: home, architecture, black & white,
     color, food, places, berlin. A photo can belong to several at once (e.g.
     a Berlin shot that's also color and architecture) — just check all that
     apply.
   - **Media type** — image or video
   - **Image**/**Video file** — upload it
   - **Alt text** — required, describes the photo for accessibility/SEO
   - **Caption** — optional short creative title
   - **Author** — optional, defaults to "X-iO"
   - **Order** — optional number; lower shows first within a category
   - Click **Publish** (top right) — drafts don't show on the live site.
4. To edit or delete: click any existing item in the list, change fields (or
   use the **⋯** menu → **Delete**), then Publish.

Changes appear on the live site within about a minute.

### Adding a lot of photos at once (bulk import)

For dozens/hundreds of images, use the bulk-import script instead of adding
them one by one. It uploads each file to Sanity and asks Gemini to write the
alt text, caption, and author in the same pass — no separate step needed.

1. Create a `media-import/` folder next to `site/` (i.e. at the same level as
   this README), with one subfolder per category:
   ```
   media-import/
     home/
     architecture/
     black-white/
     color/
     food/
     places/
     berlin/
   ```
2. Name each file `<category>_by_<author>_<number>.<ext>`, e.g.
   `black-white_by_X-iO_1000.jpg` — `<number>` only exists to keep filenames
   unique on disk (it's not the on-site sort order); `<author>` is read
   straight from the filename into the document's Author field. Photos
   (`.jpg`, `.jpeg`, `.png`, `.webp`, `.gif`) and videos (`.mov`, `.mp4`,
   `.webm`, `.m4v`) can be mixed freely. **If a file belongs to more than one
   category, copy the same file into each relevant folder** — the script
   detects it's the same file (by content, not filename) and tags it with
   all of them instead of creating duplicates.
3. In `site/.env.local`, add a write-access Sanity token (Sanity project →
   API → Tokens → Add token → **Editor** permission) and a free
   [Gemini API key](https://aistudio.google.com/apikey):
   ```
   SANITY_API_TOKEN=sk...
   GEMINI_API_KEY=...
   ```
4. From `site/`, run:
   ```bash
   npm run import-media
   ```
   Images get real alt text + a caption generated by Gemini (one request per
   image, paced 6 seconds apart to stay under the free tier's rate limit).
   Videos get a plain fallback alt/caption for now — no Gemini video
   understanding yet.
5. Check `/studio` afterward to review/adjust titles, alt text, and ordering.

**To fully replace one category** (delete every existing document tagged
with it, then re-upload everything currently in that folder) without
touching any other category:
```bash
npm run import-media -- --replace=black-white
```
Multiple categories: `--replace=black-white,berlin`.

`media-import/` is git-ignored — it's just a local staging folder, Sanity is
the actual storage.

### Backfilling alt text/captions on photos already in Sanity

If you have existing documents missing real alt text/captions (e.g. added
before Gemini generation existed, or a run that got interrupted), this
works directly against what's already uploaded — no local files needed, it
fetches a downsized copy of each image straight from Sanity's own CDN.

```bash
npm run backfill-metadata -- --title="architecture 1086"   # one photo, for testing
npm run backfill-metadata -- --id=mediaItem-xxxxxxxxxxxxxxxx
npm run backfill-metadata -- --category=architecture         # one whole collection
npm run backfill-metadata -- --category=architecture --limit=10
```
By default it only processes documents still missing real data (no caption
yet, or alt still equal to the fallback title), so it's always safe to
re-run after a partial/quota-limited run — nothing already done gets
reprocessed or re-billed against your quota. Pass `--force` to regenerate
regardless.

### Removing a whole category

```bash
npm run delete-media -- home
```
Deletes every document tagged with that category. If a document also
belongs to other categories, it's just untagged (kept, still shows on its
other galleries) instead of deleted.

Notes (apply to all three scripts above):
- **Rate limit**: Gemini calls are paced at one request per 6 seconds to
  stay under the free tier's 10 requests/minute. For a few hundred files,
  expect it to take a while — that's expected.
- If a Gemini call fails for a given file (quota, network, etc.), that file
  falls back to a plain title-based alt/caption rather than stopping the
  run — check the console output for `Gemini ... failed` warnings and
  re-run `backfill-metadata` afterward to fix just those.
